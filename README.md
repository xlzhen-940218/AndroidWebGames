# AndroidWebGames

项目定位：把静态 Web 游戏以原生 APK 形式分发，给移动用户提供类似应用内游戏目录的体验。

## 目标场景
- 离线或弱网环境下仍能启动并游玩已打包的 HTML5 游戏。 
- 在有网络时优先加载运营或远端托管的游戏目录与内容。

## 业务逻辑（用户流程）
- 启动：应用启动时会检测网络状态。
	- 有网络：优先加载远端首页（在线目录/运营页面）。
	- 无网络：回退并加载内置的本地首页（`app/src/main/assets/webgames/index.html`）。

- 首页（本地 `index.html` 的业务功能）：
	- 展示“精选游戏”列表，支持按标题/描述搜索和在“列表/宫格”两种视图间切换。
	- 游戏项点击后会打开游戏页面或链接；外部/子页面在单独的 Activity 中打开，以便区分主目录与详情页。

- 导航与交互：
	- 当展示非首页内容时，会显示“主页”按钮，用户可一键返回目录首页。
	- 返回键会首先在当前浏览历史中回退，若无历史则退出应用。

## 离线与资源注意点
- 本地离线内容位置建议：`app/src/main/assets/webgames/`（主入口为 `index.html`）。
- 当前本地首页引用了外部资源（例如通过 CDN 的图标/样式），这些在无网络时会不可用——若要保证完全离线体验，请把必要的静态资源（CSS/字体/图标/脚本）一并打包到 `assets` 中。

## 权限与运行
- 需要权限：`INTERNET`（用于加载远端内容）和 `ACCESS_NETWORK_STATE`（用于检测网络状态）。
- 构建与测试：在 Android Studio 中打开项目并运行，或使用命令行构建：
```
./gradlew assembleDebug
```

## 业务风险与建议
- 安全策略：当前实现会在某些情况下继续加载存在证书问题的页面，这在商业/生产环境中存在风险。上线前应确认 HTTPS 证书校验与安全策略。
- 离线体验优化：为保证离线时完整体验，建议将常用静态资源从 CDN 回退到本地包内。
